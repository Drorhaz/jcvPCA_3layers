Core Framework

Pipeline Specification & Framing Guardrails

This implementation architecture outlines a three-layer pipeline adapting JcvPCA to 3D motion-capture data. It must be implemented conservatively and transparently.

Scientific Framing Guardrail This system does not claim to reproduce direct encoder-based anatomical joint-angle measurements. Instead, it is strictly framed as an adaptation of the JcvPCA computational framework to Motive-derived relative joint rotation-vector features, validated by Layer 1 raw marker quality control.

Experimental Design Structure

The pipeline processes data from Gaga Task Part 1, where each participant goes through three distinct longitudinal timepoints, repeating Part 1 twice per session:

  • Timepoint 1 (Reference): T1_P1_r1, T1_P1_r2
  • Timepoint 2 (Comparison): T2_P1_r1, T2_P1_r2
  • Timepoint 3 (Comparison): T3_P1_r1, T3_P1_r2

Contextual Mapping

Gaga Task Part 1 Movement Reference Window

Movement segments compiled from the task transcript used to extract biomechanically coherent, stable time series for PCA analysis.

Implementation Rule: Why use these grouped blocks? Some individual cues are relatively short, especially the curve-related cues such as hands, elbows, shoulders, and nose. Analyzing each short cue separately may be unstable for PCA-based methods. Grouping related cues creates longer and more interpretable movement blocks.

Suggested minimum block duration: For JcvPCA-style analysis, it is preferable to use movement blocks of approximately 20–40 seconds or longer, when possible. This helps ensure that each block contains enough movement variability for PCA.

Overview

Purpose & Dataset Definition

Compares whole-body joint contribution structural changes from reference (Dataset A, fixed at T1) to comparison targets (Dataset B, tracking T2 or T3 longitudinal variance).

Mathematical Feature Adaptation The baseline paper used simulated joint trajectories of scalar variables ($joint_i, joint_j$). This architecture adapts the framework to multi-component filtered relative joint rotation-vector features ($joint_{rx}, joint_{ry}, joint_{rz}$) inside the local bone frames.

    "left_hip": ["left_hip_rx", "left_hip_ry", "left_hip_rz"]
}

selected_m: int                                # Retained PC dimensions (Calculated dynamically 
                                               # from A to satisfy e.g., 80% or 90% cumulative variance)
Step 01

Validate Inputs

Enforce strict structural filters on the input spaces prior to coordinate transformations.

  • Assert A_data and B_data contain identical feature dimensions.
  • Assert feature_names maintain perfectly matching matrix column sorting sequences.
  • Assert input columns are strictly restricted to Layer 2 output features (filtered relative rotation-vector components).
Strict Processing Rejections DO NOT allow raw marker columns, raw quaternions, global bone quaternions, or previously z-scored features into the PCA execution space.
Step 02

Center Dataset A Independently

Isolate mean coordinates of the reference dataset.

A_mean = mean(A_data, axis = samples)
A_centered = A_data - A_mean
Normalization Limitation Matches paper logic. Do not z-score, divide by standard deviation, range-normalize, or scale variance across components.
Step 03

Fit PCA on Reference Dataset A

Construct the reference principal component frame of matrix $R_A$.

PCA_A = PCA(n_components = selected_m)
PCA_A.fit(A_centered)

R_A = PCA_A.components_                       # Shape: [selected_m, n_features]
explained_variance_ratio_A = PCA_A.explained_variance_ratio_
Step 04

Center Dataset B Independently

Center the comparison tracking set to its individual local frame origin.

B_mean = mean(B_data, axis = samples)
B_centered = B_data - B_mean
Crucial Deviation Warning This calculation must NOT use B_data - A_mean. Dataset B must step into its own zero-centered coordinate frame before being projected into A's framework.
Step 05

Project B into A's PCA Space

Map the independently centered data matrix of B onto the reference vectors of A.

# Safe Manual Matrix Multiplication
B_scores_in_A_space = B_centered @ transpose(R_A)   # Shape: [n_samples_B, selected_m]
Library Architecture Warning Avoid using automated blackbox library calls like PCA_A.transform(B_raw) if the backend pipeline implicitly recalculates and subtracts A's mean vector from B.
Step 06

Run PCA on Projected B Space

Compute internal variance directions of B inside the coordinates defined by A.

PCA_B_in_A_space = PCA(n_components = selected_m)
PCA_B_in_A_space.fit(B_scores_in_A_space)

R_B_in_A_space = PCA_B_in_A_space.components_       # Shape: [selected_m, selected_m]
Step 07

Reproject B Loadings into Original Feature Space

Transform B's local subspace orientations back into native relative rotation features.

B_reprojected_loadings = R_B_in_A_space @ R_A       # Shape: [selected_m, n_features]
B_abs_loadings = abs(B_reprojected_loadings)
Step 08

Compute A Absolute Loadings

Establish structural base magnitude benchmarks from reference matrix configurations.

A_loadings = R_A
A_abs_loadings = abs(A_loadings)                    # Shape: [selected_m, n_features]
Step 09

Compute Axis-Level JcvPCA

Examine directional deviations across all component vectors.

JcvPCA_axis = B_abs_loadings - A_abs_loadings
Interpretation Guide
  • Positive Value: Feature component vector demonstrates increased contribution weight in Dataset B relative to reference Dataset A.
  • Negative Value: Feature component vector demonstrates diminished contribution weight in Dataset B relative to reference Dataset A.
Step 10

Compile Axis-Level JRW Table

Consolidate structural analytical properties across metrics into standardized row formats.

for each pc_index in selected_m:
    for each feature_name in feature_names:
        emit_row(
            pc_index, feature_name, joint_name, component,
            A_loading, B_reprojected_loading, A_abs_loading, B_abs_loading,
            JcvPCA_axis, explained_variance_ratio_A
        )

export_to_csv("JRW_axis_table.csv")
export_to_csv("JcvPCA_axis_table.csv")
Step 11

Group Coordinate Axes into Joint-Level Values

Consolidate directional $rx, ry, rz$ vectors into Euclidean representations using root-sum-square calculations.

for each joint in joint_map:
    for each pc in selected_m:
        # Extract axis components
        A_rx, A_ry, A_rz = get_loadings(A_abs_loadings, joint, pc)
        B_rx, B_ry, B_rz = get_loadings(B_abs_loadings, joint, pc)
        
        # Root-Sum-Square (RSS) transformation
        JRW_A_joint = sqrt(A_rx^2 + A_ry^2 + A_rz^2)
        JRW_B_joint = sqrt(B_rx^2 + B_ry^2 + B_rz^2)
        
        # Calculate Joint JcvPCA delta
        JcvPCA_joint = JRW_B_joint - JRW_A_joint
Custom Modification Rationale This RSS step is a designated structural extension accommodating 3D rotation vectors. The source literature utilizes pre-aggregated scalar metrics, bypassing this multi-dimensional consolidation phase.
Step 12

Optional Explained-Variance Weighting

Scale components by their relative importance within reference conditions.

for each pc in selected_m:
    Weighted_JcvPCA_axis = JcvPCA_axis * explained_variance_ratio_A[pc]
    Weighted_JcvPCA_joint = JcvPCA_joint * explained_variance_ratio_A[pc]
Retention Rule Retain both configurations. Unweighted arrays protect narrow component-specific resolution, while weighted parameters provide broad holistic summaries.
Step 13

Natural Variability Wrapper

Calibrate system observations against localized timepoint noise profiles using baseline repetitions ($r1$ vs $r2$).

# Baseline Natural Variability Target (Group 4)
A_var = T1_P1_r1_Group4
B_var = T1_P1_r2_Group4

Natural_Variability_Group4 = Compute_JcvPCA(A_var, B_var)
Terminology Restriction & Interpretation A change should be interpreted as more meaningful if it exceeds the within-baseline repetition variability, appears in task-relevant PCs, and shows a consistent direction.

Categorize longitudinal changes explicitly as:
  • "increased beyond T1 repetition-level variability"
  • "decreased beyond T1 repetition-level variability"
  • "within T1 repetition-level variability / inconclusive"
Do not state that findings are "statistically significant" unless formal hypothesis testing frameworks are explicitly appended later.
