# Layer 2 — Pipeline outputs

Generated artifacts from Layer 2 kinematic processing runs. Each session gets a folder named after the input file with stage subfolders (`00_csv_structure/` through `08_filtered_rotvecs/`).

## What is included in Git

Small reports and summaries that help reviewers evaluate pipeline behavior:

- `report.md`, `assumptions_and_limitations.md`, `assumptions_log.md`
- Summary CSVs (joint maps, QC summaries, filter diagnostics, index files)
- `layer2_exports/*/layer2_session_report.md`, `layer2_session_summary.json`
- Archive index files (`stage00_01_report_index.md`, etc.)

## What is excluded from Git

Large regenerated intermediate data:

| Pattern | Typical size | Reason |
|---------|-------------|--------|
| `*.parquet` | 50–145 MB | Full frame × joint matrices |
| `relative_quaternions.csv` | 250–300 MB | Stage 06 frame-level output |
| `relative_rotation_vectors.csv` | 400–470 MB | Stage 07 frame-level output |
| `filtered_relative_rotation_vectors.csv` | 700–780 MB | Stage 08 frame-level output |
| `global_quaternions_sign_continuous.csv` | 115–130 MB | Stage 05 frame-level output |

These are excluded via the root `.gitignore` and can be regenerated by re-running the pipeline.

## Reference session

The primary development session is **`671_T1_P1_R1`**. Its stage reports are fully included; only the large frame-level CSV/parquet files are omitted.

## How to regenerate

```bash
cd Layer2_Motive_Kinematics
source .venv/bin/activate
layer2-motive run-until --input /path/to/motive.csv --stage 08 \
  --output-dir outputs/session_name
```

Final Layer 2.5 handoff artifact: `layer2_exports/{session}/layer2_session_filtered_rotvecs.parquet` (generated locally, not in Git).
